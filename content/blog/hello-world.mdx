## Welcome

This is your first blog post! It's written in MDX, which means you can use React components alongside your Markdown content.

## Getting Started

To customize this boilerplate for your SaaS:

1. Edit `config/site.ts` with your app's name, description, and branding
2. Set up your environment variables (see `.env.example`)
3. Deploy to Vercel

## Code Examples

Here's a quick example of how the central config works:

```typescript
import { siteConfig } from "@/config/site";

// Access your app's configuration anywhere
console.log(siteConfig.name); // "Your App Name"
console.log(siteConfig.theme.primary); // "#6366f1"
```

## Features

This boilerplate includes:

- **Landing Page** — dub.co-inspired design with configurable sections
- **Authentication** — Google OAuth via Auth.js
- **Database** — Prisma + Neon Serverless Postgres
- **Billing** — Optional Paddle integration
- **Blog** — MDX-based blog (you're reading it!)
- **SEO** — Auto-generated metadata, sitemap, and OG images
- **Dark Mode** — System preference with manual toggle
- **i18n** — Ready for internationalization
